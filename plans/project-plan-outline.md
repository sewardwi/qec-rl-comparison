# RL for Quantum Error Correction: Implementation Plan

## 1. Overview

This project compares two RL decoders (DQN and PPO) against the standard Minimum-Weight Perfect Matching (MWPM) decoder for surface code quantum error correction. The emphasis is on rich quantum simulation — diverse noise models, circuit-level analysis, syndrome visualization — while implementing clean, well-compared RL algorithms.

**Key libraries**: Stim (circuit simulation), PyMatching (MWPM baseline), Sinter (parallelized benchmarking), Stable-Baselines3 (DQN/PPO), Gymnasium (RL environment).

**Key decision**: We build a custom Gymnasium environment rather than using `gym-surfacecode` (Sweke et al.), which is hard-limited to distance-5. Our environment wraps Stim directly and supports arbitrary distances and noise models.

## 3. Quantum Simulation Modules (Emphasis Area)

### 3.1 `src/quantum/surface_code.py` — Circuit Generation & Sampling

The foundation module. Wraps Stim's circuit generation with structured metadata.

**Key types:**

```python
@dataclasses.dataclass(frozen=True)
class SurfaceCodeParams:
    distance: int          # 3, 5, or 7
    rounds: int            # typically equal to distance
    basis: str = "Z"       # "X" or "Z" memory experiment

    @property
    def num_data_qubits(self) -> int:
        return self.distance ** 2

    @property
    def num_stabilizers(self) -> int:
        return self.distance ** 2 - 1

    @property
    def task_string(self) -> str:
        return f"surface_code:rotated_memory_{self.basis.lower()}"


@dataclasses.dataclass
class SurfaceCodeCircuit:
    params: SurfaceCodeParams
    noise_params: dict[str, float]
    circuit: stim.Circuit
    # Lazily compiled sampler, cached for reuse across episodes
    _sampler: stim.CompiledDetectorSampler | None = None
```

**Key functions:**

| Function | Purpose |
|---|---|
| `generate_circuit(params, noise_params)` | Generate a noisy Stim circuit via `stim.Circuit.generated()` |
| `SurfaceCodeCircuit.sample(shots)` | Batch-sample `(syndromes, observables)` via compiled detector sampler |
| `SurfaceCodeCircuit.detector_error_model` | Property returning the DEM with `decompose_errors=True` |
| `get_detector_coordinates()` | Return detector index → `(x, y, t)` coordinate mapping |
| `get_qubit_layout(params)` | Compute data qubit and stabilizer positions on the rotated lattice |
| `analyze_circuit_structure(circuit)` | Return gate counts, depth, qubit counts, DEM size |

**Stim API usage:**

```python
# Generate circuit
circuit = stim.Circuit.generated(
    "surface_code:rotated_memory_z",
    distance=5, rounds=5,
    after_clifford_depolarization=0.01,
    before_round_data_depolarization=0.01,
    before_measure_flip_probability=0.01,
    after_reset_flip_probability=0.01,
)

# Sample syndromes + observables
sampler = circuit.compile_detector_sampler(seed=42)
syndromes, observables = sampler.sample(shots=1024, separate_observables=True)
# syndromes: bool array (1024, num_detectors)
# observables: bool array (1024, num_observables)
```

**Design note:** The sampler is compiled once and cached because recompilation is expensive. Each environment reset draws from a pre-sampled batch of 1024 shots for efficiency.

### 3.2 `src/quantum/noise_models.py` — Noise Model Library

Six physically-motivated noise models, each with a clear physical justification:

| # | Model | Physical Motivation | Implementation |
|---|---|---|---|
| 1 | **Depolarizing** | Standard benchmark; equal X, Y, Z error probability | Pass `p` to all four Stim noise params |
| 2 | **Biased-Z** | Dephasing-dominated (superconducting qubits); Z errors η× more likely than X,Y | Custom circuit: inject `PAULI_CHANNEL_1` with biased probabilities after each gate |
| 3 | **Biased-X** | Bit-flip-dominated | Same technique as biased-Z, rotated |
| 4 | **Measurement-dominated** | Readout errors ≫ gate errors | Set `before_measure_flip_probability = p`, all others `= p/10` |
| 5 | **Asymmetric gate** | Two-qubit gates 10× noisier than single-qubit | Custom circuit: different `DEPOLARIZE1` vs `DEPOLARIZE2` rates |
| 6 | **Correlated** | Spatially correlated errors (crosstalk) | Use Stim's `CORRELATED_ERROR` for nearby qubit pairs |

**Key types:**

```python
class NoiseModelType(Enum):
    DEPOLARIZING = "depolarizing"
    BIASED_Z = "biased_z"
    BIASED_X = "biased_x"
    MEASUREMENT = "measurement"
    ASYMMETRIC_GATE = "asymmetric_gate"
    CORRELATED = "correlated"

@dataclasses.dataclass
class NoiseModelConfig:
    model_type: NoiseModelType
    physical_error_rate: float
    bias_ratio: float = 1.0            # For biased models: η
    measurement_scale: float = 1.0     # For measurement model
    correlation_length: float = 0.0    # For correlated model
```

**Implementation approach for biased/correlated noise:**
1. Generate a noiseless circuit (`stim.Circuit.generated(...)` with all noise params = 0)
2. Iterate through circuit instructions
3. After Clifford gates, insert custom `PAULI_CHANNEL_1` or `PAULI_CHANNEL_2` with bias-adjusted probabilities
4. For correlated noise, add `CORRELATED_ERROR` instructions for nearby qubit pairs

### 3.3 `src/quantum/syndrome.py` — Syndrome Processing & Visualization

Converts Stim's flat detector output into spatially meaningful representations for CNN input and analysis.

**Key functions:**

| Function | Purpose |
|---|---|
| `reshape_syndrome_to_grid(flat, coords, d, rounds)` | Flat bool array → `(rounds, grid_h, grid_w)` spatial grid |
| `reshape_syndrome_batch(flat_batch, ...)` | Batch version: `(batch, num_detectors)` → `(batch, rounds, h, w)` |
| `compute_syndrome_diff(grid)` | XOR between consecutive rounds → `(rounds-1, h, w)` |
| `get_syndrome_statistics(syndromes, d)` | Detection rate, spatial/temporal correlations, cluster sizes |
| `visualize_syndrome(grid, d, round_idx)` | Plot one syndrome round on the lattice |
| `visualize_syndrome_spacetime(grid, d)` | 3D space-time visualization across all rounds |

**Grid dimensions by distance:**
- d=3: syndrome grid ≈ 5×5 (from detector coordinate ranges)
- d=5: syndrome grid ≈ 9×9
- d=7: syndrome grid ≈ 13×13

The grid mapping is derived from `circuit.get_detector_coordinates()`, which returns `(x, y, t)` for each detector. Unique `(x, y)` positions define the spatial grid; `t` indexes the round.

### 3.4 `src/quantum/decoder_baseline.py` — MWPM Baseline

```python
class MWPMDecoder:
    @classmethod
    def from_circuit(cls, circuit: stim.Circuit) -> MWPMDecoder:
        dem = circuit.detector_error_model(decompose_errors=True)
        matching = pymatching.Matching.from_detector_error_model(dem)
        return cls(matching)

    def decode_batch(self, syndromes: np.ndarray) -> np.ndarray:
        return self._matching.decode_batch(syndromes)

    def evaluate(self, syndromes, observables) -> dict:
        predictions = self.decode_batch(syndromes)
        errors = np.any(predictions != observables, axis=1)
        return {"logical_error_rate": errors.mean(), ...}
```

Expected threshold: ~1% for depolarizing noise with circuit-level noise on the rotated surface code.

### 3.6 `src/quantum/sinter_benchmarks.py` — Parallelized Monte Carlo

Uses Sinter for high-throughput MWPM baseline generation:

```python
def run_benchmark(tasks, num_workers=4, max_shots=100_000, max_errors=1000):
    return sinter.collect(
        num_workers=num_workers,
        tasks=tasks,
        max_shots=max_shots,
        max_errors=max_errors,
        decoders=["pymatching"],
    )
```

This generates the gold-standard baseline curves (logical error rate vs. physical error rate for each distance) that RL decoders are compared against. Sinter handles parallelization, early stopping (after `max_errors` logical errors), and result aggregation.

## 4. Gymnasium Environment

### 4.1 `src/envs/surface_code_env.py` — MDP Formulation

**State (observation):**
- Shape: `(num_channels, grid_h, grid_w)` as `float32`
- Channels (in "stacked rounds" mode):
  - Channels 0 to `rounds-1`: syndrome grids for each round (binary, 0/1)
  - Channel `rounds`: X-correction history (which data qubits have X corrections)
  - Channel `rounds+1`: Z-correction history
- Total channels = `distance + 2` (since `rounds = distance`)

**Actions:**
- `Discrete(3 * d² + 1)` where `d` = code distance
  - Actions `0` to `d²-1`: apply X correction on data qubit `i`
  - Actions `d²` to `2d²-1`: apply Z correction on data qubit `i - d²`
  - Actions `2d²` to `3d²-1`: apply Y correction on data qubit `i - 2d²` (toggles both X and Z)
  - Action `3d²`: **commit** — finalize corrections for the current round; advance to next round or end episode

| Distance | Data qubits | Action space size |
|---|---|---|
| d=3 | 9 | 28 |
| d=5 | 25 | 76 |
| d=7 | 49 | 148 |

**Reward:**
- Sparse: `+1` if logical qubit preserved after all rounds, `-1` otherwise
- Determined by checking whether the net correction's effect on the logical observable matches the actual observable from Stim

**Episode flow:**
1. `reset()`: draw next pre-sampled syndrome shot; reset correction history
2. Agent sees round 0 syndrome, applies corrections (X/Z/Y on individual qubits)
3. Agent takes "commit" action → advance to round 1
4. Repeat until all `d` rounds are processed
5. Episode terminates; reward computed

**Efficiency design — pre-sampling:**
- On initialization (and whenever the batch is exhausted), sample 1024 `(syndrome, observable)` pairs from Stim in one call
- Each `reset()` simply advances an index into this batch
- The agent's corrections do NOT re-simulate the circuit; they are bookkeeping only
- The observable from Stim tells us the ground truth, and we check if corrections are consistent
- This is physically correct: in real QEC, the decoder sees syndromes and issues corrections without affecting the quantum state

**Reward computation:**
```
correction_flips_logical = parity of X-corrections along the logical operator support
actual_flip = observable[0] from Stim
success = (correction_flips_logical == actual_flip)
reward = +1.0 if success else -1.0
```

For `rotated_memory_z`: the logical Z operator runs along a column of data qubits. An X correction on any qubit in that column flips the logical observable.

### 4.3 `src/envs/reward_shaping.py`

| Strategy | Description | Policy-preserving? |
|---|---|---|
| `sparse` | +/-1 at episode end | Yes (ground truth) |
| `syndrome_reduction` | Small positive for reducing syndrome weight | No (heuristic) |
| `potential_based` | `F(s') - F(s)` where `F = -syndrome_weight` | Yes (provably) |
| `matching_guided` | Reward for agreeing with MWPM suggestions | No (teacher-guided) |

Default is `sparse`. Use `potential_based` if training is too slow — it accelerates early learning without changing the optimal policy.

## 5. RL Agents

### 5.1 `src/agents/networks.py` — CNN Feature Extractors

**Why custom CNNs?** The syndrome grids are tiny (5×5 to 13×13). The standard Atari-style Nature CNN uses 8×8 and 4×4 kernels, which would collapse these grids to nothing. We use 3×3 kernels with `padding=1` (preserves spatial dimensions) and **no pooling**.

**`SyndromeCNN`** (default):
```
Conv2d(in_channels, 32, 3, padding=1) → ReLU
Conv2d(32, 64, 3, padding=1) → ReLU
Conv2d(64, 64, 3, padding=1) → ReLU
Flatten → Linear(64*h*w, 128) → ReLU
```

Extends SB3's `BaseFeaturesExtractor` so it plugs directly into DQN/PPO policy kwargs.

**`ResidualSyndromeCNN`** (for d=7 if needed):
```
Conv2d(in_channels, 32, 3, padding=1) → BN → ReLU
ResBlock(32 → 64)
ResBlock(64 → 64)
ResBlock(64 → 128)
Global Average Pooling → Linear(128, 128) → ReLU
```

Skip connections help gradient flow for the deeper network needed at d=7.

### 5.2 `src/agents/dqn_agent.py`

Wraps SB3's `DQN` with `SyndromeCNN`:

```python
def create_dqn_agent(env, **kwargs) -> DQN:
    policy_kwargs = dict(
        features_extractor_class=SyndromeCNN,
        features_extractor_kwargs=dict(features_dim=128),
        net_arch=[256, 256],
    )
    return DQN("CnnPolicy", env, policy_kwargs=policy_kwargs, ...)
```

**Default hyperparameters:**

| Parameter | Value | Rationale |
|---|---|---|
| `learning_rate` | 1e-4 | Standard for DQN with CNNs |
| `buffer_size` | 100,000 | Moderate; observations are small |
| `batch_size` | 64 | Standard |
| `gamma` | 0.99 | Episodes are short (~d steps) but slight discounting for commit timing |
| `exploration_fraction` | 0.3 | High; agent needs to explore many correction patterns |
| `exploration_final_eps` | 0.05 | Standard |
| `target_update_interval` | 1,000 | Standard for stable Q-learning |
| `train_freq` | 4 | Update every 4 steps |
| `total_timesteps` | 500K (d=3), 1M (d=5), 2M (d=7) | Scale with complexity |

### 5.3 `src/agents/ppo_agent.py`

Wraps SB3's `PPO` with shared CNN between actor and critic:

```python
def create_ppo_agent(env, **kwargs) -> PPO:
    policy_kwargs = dict(
        features_extractor_class=SyndromeCNN,
        features_extractor_kwargs=dict(features_dim=128),
        net_arch=dict(pi=[256, 256], vf=[256, 256]),
        share_features_extractor=True,
    )
    return PPO("CnnPolicy", env, policy_kwargs=policy_kwargs, ...)
```

**Default hyperparameters:**

| Parameter | Value | Rationale |
|---|---|---|
| `learning_rate` | 3e-4 | Standard PPO |
| `n_steps` | 2,048 | Enough data per update since episodes are short |
| `batch_size` | 64 | Standard |
| `n_epochs` | 10 | Standard PPO |
| `gamma` | 0.99 | Same as DQN for fair comparison |
| `gae_lambda` | 0.95 | Standard |
| `clip_range` | 0.2 | Standard PPO clipping |
| `ent_coef` | 0.01 | Entropy bonus prevents collapsing to always "commit" |
| `total_timesteps` | 500K (d=3), 1M (d=5), 2M (d=7) | Same budget as DQN for fair comparison |

### 5.4 `src/agents/callbacks.py`

- **`DecodingEvalCallback`**: Every `eval_freq` steps, run agent on 1000 episodes, compute logical error rate, compare to MWPM baseline, log ratio to TensorBoard, save best model
- **`CurriculumCallback`**: Optional; linearly increase physical error rate from `p_start` to `p_end` over training to help the agent learn basic patterns first

## 6. Evaluation Pipeline

### 6.1 `src/evaluation/evaluator.py`

Runs all decoders across a grid of parameters:

```python
class DecoderEvaluator:
    def __init__(self, distances, error_rates, noise_models, num_shots=10_000): ...
    def add_decoder(self, name, decode_fn): ...
    def run(self) -> pd.DataFrame: ...
    # Returns DataFrame: decoder | distance | p_phys | noise_model | logical_error_rate | ci_lower | ci_upper
```

**Evaluation grid:**
- Distances: [3, 5, 7]
- Physical error rates: `np.logspace(-3, -0.5, 20)` (0.001 to ~0.316)
- Noise models: depolarizing, biased_z, biased_x, correlated, measurement
- Shots per point: 10,000 (adjustable)

For RL decoders, the `decode_fn` wrapper loads the trained SB3 model and runs episodes to completion, recording success/failure.

## 10. Key Design Decisions

### Pre-sampling vs. on-the-fly simulation
The environment pre-samples batches of 1024 `(syndrome, observable)` pairs from Stim. Agent corrections are bookkeeping only — they don't re-simulate the circuit. This is physically correct (real decoders can't affect quantum state) and ~1000× faster than per-step simulation.

### Action space: flat vs. factored
We use a flat `Discrete(3d²+1)` action space rather than factored (choose qubit, then choose Pauli). The flat space is simpler to implement with SB3 and works well for d=3,5. At d=7 (148 actions), DQN may struggle — this is itself an interesting finding.

### CNN architecture: no pooling, small kernels
Standard Atari-style CNNs destroy spatial information on grids this small. Our 3×3 kernels with `padding=1` preserve spatial dimensions through all layers. No max-pooling. This is critical.

### Shared vs. separate feature extractors (PPO)
PPO shares the CNN between actor and critic (`share_features_extractor=True`) to reduce computation and stabilize learning. The syndrome features are useful for both policy and value prediction.

### Reward: sparse by default
Sparse +/-1 is the honest signal — it directly measures decoding success. Potential-based shaping (`F(s') - F(s)` with `F = -syndrome_weight`) is a provably policy-preserving fallback if training stalls.

## 11. Expected Results

1. **Threshold curves (depolarizing noise)**: MWPM threshold ≈ 1%. RL decoders should approach but likely slightly underperform MWPM at d=3. Gap may widen at d=5, d=7 due to larger action spaces.

2. **DQN vs. PPO**: PPO expected to be more sample-efficient due to entropy bonus preventing premature convergence to "always commit." DQN may achieve lower final error rate due to off-policy replay enabling more data reuse.

3. **Biased noise**: This is where RL can potentially beat MWPM. MWPM assumes symmetric error weights; RL agents trained on biased noise can adapt to the asymmetry. If this works, it's the strongest result.

4. **Correlated noise**: MWPM assumes independent errors. RL decoders may handle correlations better, though this is speculative.

5. **Scaling**: If d=7 proves intractable for RL, present d=3,5 results with analysis of why scaling is hard (action space, episode length, credit assignment).
